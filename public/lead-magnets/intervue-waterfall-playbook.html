02

The waterfall

Every account tested top-to-bottom, routed to the first tier it matches.

first match wins 
1. IF hiring_engineers is FALSE -> Not currently in market (No open technical role means no live interview load to relieve. Hold for re-scan.)
2. IF interview_vendor_detected is not none -> Tier 1, Competitor Displacement (They already buy outsourced interviews, so the only open question is a better, faster, cheaper vendor.)
3. IF else, assessment_tool_detected is not none -> Tier 2, Screen Automated, Live Round Manual (They automated the screen but every live round still lands on a senior engineer's calendar.)
4. IF else, open_tech_roles is 10 or more -> Tier 3, High-Volume Bleed (High req volume with no tool means engineers are running every round by hand, the biggest raw pain.)
5. IF else, open_tech_roles is 1 to 9 -> Tier 4, Scaling Team, No System (Actively hiring at small scale, every interview pulls a founder or senior engineer off building.)
6. IF else (target-fit, not posting, or unreadable) -> Catch-all, Next Wave / Bench Load (The generic, cyclical pain Intervue was built for. Broadest, softest, catches everything above missed.)

03

04

05

Weeks 6-12: Tiers 3, 4 and the catch-all phase in. Reassess the full cascade at week 12.

06

Ongoing: the strategy is not rebuilt. The lead agent keeps adding accounts to the same five tiers. Add leads forever.

Scale / cut rules

1 positive reply per 150 contacts Perfect. Scale the tier hard.
1 per 300 Good. Let it run untouched.
1 per 450-500 Borderline. Test one copy variant, vary the lead magnet first.
Worse than 1 per 500, or reply rate under 1% after 1,500+ sends Dead. Cut and rework, don't tinker.
