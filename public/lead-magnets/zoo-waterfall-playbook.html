Signal

The buying signal is the CAD tool they hire for

A company's vertical changes the noun in the email. The CAD tool named in its job post decides the campaign it lands in.

strongest signal
01

A job post that names a CAD tool proves the company does CAD work today, against a real headcount budget.

02

The named tool reveals the exact pain: a legacy seat implies license cost, Windows lock-in and slow parametric iteration.

03

Actively hiring a design engineer means the team is scaling, which is precisely when onboarding time and throughput hurt most.

04

Desktop CAD leaves no web footprint, so the tool named in a job post is the only reliable at-scale intent signal. We invert on presence, never on absence.

05

Every hardware company iterates eventually, so no account is truly cold. The catch-all sweeps the rest.

Waterfall

One cascade routes the whole hardware market

Every hardware account flows top to bottom: legacy-seat payer, modern-cloud team, scaling and tool-agnostic, or catch-all. Most land in exactly one.

first match wins 
1. IF hiring a design role AND JD names SolidWorks / Creo / NX / CATIA -> Tier 1 (Paying premium legacy seats while scaling)
2. IF hiring a design role AND JD names Fusion 360 / Onshape -> Tier 2 (Modern tool, but still modeling by hand)
3. IF hiring a mechanical / hardware / ID engineer, no CAD tool named -> Tier 3 (Design throughput is the bottleneck)
4. IF any remaining hardware company in TAM -> Catch-all (Next iteration, legacy CAD is the slow step)
